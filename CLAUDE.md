## Project Summary

A cross-platform desktop pet for macOS and Windows. The user grows a pet from a starter form through evolution stages by being present and being productive. The pet loves them unconditionally; growth is earned through productivity but affection is never conditional on it.

The product exists in memory of a cat named Meowzers (see design doc). Every design decision is in his honor.

**Build strategy:** Radically scoped v1. Two personalities (cuddly + powerful), two evolution stages (1st + 2nd), no legacy stage yet. See `desktop-pet-design.md` § Build Order Recommendations for full v1 scope and version roadmap.

## Tech Stack

- **Tauri v2** (Rust backend + web frontend) — cross-platform desktop framework
- **Frontend:** React + TypeScript (Vite for dev tooling)
- **Persistence:** SQLite via `rusqlite` (direct, not `tauri-plugin-sql`)
- **State management:** Local React state. No global store. This is not a complex SPA.
- **Animation:** CSS animations + `requestAnimationFrame` for the focus timer. No heavyweight animation libraries.
- **Drag-and-drop:** `@dnd-kit/core` + `@dnd-kit/sortable` + `@dnd-kit/utilities` (for task reordering only)
- **Notifications:** `tauri-plugin-notification` (native OS notifications)
- **System tray:** Tauri's tray API
- **Build / CI:** GitHub Actions for cross-platform builds

**Do not introduce:**
- Electron (use Tauri)
- Heavy state management libraries (Redux, MobX, Zustand) — this app genuinely doesn't need them
- Backend services or cloud sync in v1 — local-only
- Browser storage APIs (`localStorage`/`sessionStorage`) — use SQLite

## Architecture Principles

These are non-negotiable. If a proposed change violates one, push back explicitly.

### 1. Event-driven, never polling

- The pet's state is computed on demand from timestamps, not maintained by a running loop
- Between events, the app does effectively nothing — no tick loops, no `setInterval` polling state
- On app open: read timestamps, compute current state, render
- On wake from sleep: recompute from timestamps, never trust that timers fired correctly
- The focus timer uses `requestAnimationFrame` and updates state only once per visible second. No `setInterval`.

### 2. State is sacred

- All saves use atomic SQLite transactions
- Schema versioning is mandatory; every schema change gets a migration function
- Never break existing saves — migrations are additive, not destructive

### 3. Performance discipline

- Animations only run when they need to
- When window is hidden, RAF loops pause via `document.visibilitychange`
- Timer continues correctly on return by accounting for hidden-time as pause-time
- The app should be "always there" but doing effectively nothing 99% of the time

### 4. Cross-platform from day one

- Test both platforms on CI for every meaningful change
- File paths use Tauri's `path` API, never hardcoded
- macOS-specific config (`macOSPrivateApi: true` for transparency) lives in `tauri.conf.json`

### 5. Privacy and ownership

- All user data is local. No telemetry in v1.
- If telemetry is ever added, must be anonymous, minimal, and opt-in.
- User data is the user's. Export must be easy.

## Coding Conventions

- **TypeScript strict mode.** No `any` without a comment explaining why.
- **Rust: standard `cargo fmt` + `clippy`.** Address warnings, don't suppress.
- **Names descriptive over short.** `computeCurrentHunger()` not `getHunger()`.
- **Comments explain *why*, not *what*.** Code shows what; comments justify non-obvious choices.
- **No commented-out code.** Delete it. Git remembers.
- **Atomic transactions** for any multi-write DB operation. No partial writes.
- **Silent diminishing returns, visible warmth.** Anti-cheat and limits are internal; user-facing copy is always warm.

## What This App Is *Not*

Easy mistakes. Don't make them.

- **Not a Tamagotchi clone.** The pet does not die. No permadeath. No harsh decay.
- **Not a productivity surveillance tool.** Growth is tied to productivity; love is not. The pet loves the user regardless of performance.
- **Not an engagement-maximizing app.** No streaks that punish breaks. No FOMO. No daily login bonuses. No guilt notifications.
- **Not an enterprise product.** No manager dashboards. No team analytics. No employee monitoring. Rejected explicitly.
- **Not a wellness/therapy app.** Companion, not clinician. Don't make medical or psychological claims.
- **Not a freemium grind.** Core experience is never gated by payment. Optional cosmetics or supporter tiers OK; love is not.
